Background

The problem with phone calls

LWF Corp provides advanced AI-powered sensing technology to monitor and manage water systems — detecting leakages, unauthorized drilling, and water theft in real time. But when the system detects an alert, the process for responding was dangerously outdated.

Corporate employees had to assign tasks to field technicians via phone calls. No digital record. No real-time tracking. No accountability. The mission: design a feature that eliminates this entirely.

Competitive Analysis

How other systems handle alert assignment

We reviewed how leading task and alert management systems handle assignment workflows — to identify patterns worth adopting and pitfalls to avoid.

ClickUp
  • Action performed within the task — convenient and intuitive
  • Assignment is optional — user is not required to assign
  • Cannot provide comments during assignment
  • Cannot set deadlines throughout the assignment flow
Zendesk
  • Assignment contained within the ticket — logical and expected
  • Assigning a ticket is optional for the user
  • User-friendly and easy to understand
  • Additional assignment features are clearly visible — not hidden
Logz.IO
  • Assign alert from a dedicated screen — good separation of concerns
  • Alert assignment interface can become cluttered quickly
  • The "assign" function is hidden behind creating an alert — poor discoverability
  • Not user-friendly especially for new users
The Solution

Direct Alert Assignment & Tracking

The Direct Alert Assignment and Tracking System enables operators to assign alerts directly within the system — streamlining the process by eliminating the need for phone calls. This feature provides real-time tracking of alert status, allowing operators to monitor and control the progress of field technicians from assignment to resolution.

The goal was to create a clear, simple, and comprehensible system that eases daily tasks for corporate employees while enhancing overall operational efficiency — a feature that enables employees to digitally assign alerts to technicians, ensuring real-time tracking, accountability, and improved communication.
Key Screens: Operator

Operator experience

Map & Alert Screens

The initial state where the alert appeared alone without additional information. We added the alert status, an indication of which technicians the alert is assigned to, and an option for more details.

Alert Detail & Assign CTA

When the user clicks on an alert, they receive detailed information. We added a primary CTA button "Assign Alert" with an option for more information about the alert — making the next step unmissable.

Assign Alert — Confirmation State

The modal allows the user to assign a specific alert, prioritize it, set a deadline, and add additional information. Once confirmed, 3 assignees are added to the drilling alert — visible to all stakeholders in real time.

Key Screens: Technician

Technician mobile experience

Full lifecycle management in the field

When a technician is assigned to an alert, the mobile experience gives them everything they need to manage it end-to-end — without a single phone call:

  • ✓View full alert details and context
  • ✓Approve or deny the assignment
  • ✓Update status at each stage
  • ✓Add notes and set priorities
  • ✓Upload documentation and photos
  • ✓Track progress and communicate with the team
  • ✓Close the alert with a final report
Design System

Building for consistency

A unified design system ensured consistency and coherence across both the operator web platform and the technician mobile app — reducing redundancy, accelerating iterations, and aligning the team with one shared visual language.

Process

Collaboration & whiteboarding

Daily meetings with the UX designer and product manager ensured alignment throughout. We mapped out user flows together, with extensive feedback and iteration to refine our approach — this collaborative effort ensured the final product addressed both user needs and business goals.

My work on LWF focused on transforming the alert assignment process to enhance efficiency and transparency. Initially, alerts appeared without any additional information. Now, users can click on an alert to access comprehensive information, assign it with priorities and deadlines, and track it all the way to resolution — without a single phone call.
